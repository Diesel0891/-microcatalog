2. WHAT v1 CANNOT SOLVE (Zero-Backend Limitation) ❌

GapWhy BlockedImpact
Subscription billingNo auth = no user identityCannot monetize recurring revenue
Server-side feature gatesAll checks are client-sideUsers can bypass `is_pro` flags in console

6. BACKEND DECISION FRAMEWORK

ScenarioActionEffort
Stay zero-backend, manual premiumYou review Flutterwave dashboard, flip `is_pro` manually in Supabase1–2 weeks
Add Supabase Edge FunctionsWebhook auto-activation, server-side limit enforcement3–4 weeks
Full v2.0 backend rewriteAuth, adapters, multi-tenant, OG meta, typed errors2–3 months

Recommendation: Manual premium first (validate demand), then Edge Functions (automate), then full rewrite only at 100+ paying sellers.

---

7. KEY PRINCIPLES

1. Not a Shopify competitor — preserve conversational simplicity; add just enough structure
2. Zero backend until validated — phone-first build constraint (Mate 9/A03, 3GB RAM)
3. Third-party over custom — Flutterwave for payments, Supabase for data, Cloudinary for images
4. Additive only — every feature must be removable without breaking core journeys
5. Free tier builds habit — generous enough to demonstrate value; premium captures revenue
